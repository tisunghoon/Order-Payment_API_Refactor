### STEP 7 — PaymentService 핵심 수정

| 항목 | 내용 |
|---|---|
| **파일** | `backend/src/main/java/com/myfave/api/domain/payment/service/PaymentService.java` |
| **작업 목록** | |
| ① | `ConfirmContext` record에 `List<Long> sortedProductIds` 추가 |
| ② | `validateForConfirm` — OrderItem 조회 후 productId ASC 정렬해 Context에 적재 |
| ③ | `confirmPayment` — `self.decreaseStockForConfirm()` 직접 호출 → `lockManager.executeWithLock(keys, ..., () -> self.decreaseStockForConfirm(...))` 으로 교체 |

| ⑤ | `CancelContext` record에 `List<Long> sortedProductIds` 추가 |
| ⑥ | `validateForCancel` — productId ASC 정렬해 Context에 적재 |
| ⑦ | `cancelPayment` — 전액 취소 시 `lockManager.executeWithLock` 으로 감싸기 (부분취소는 락 불필요) |
| ⑧ | `applyCancelResult` — `findByIdForUpdate` → `findByProductIdAndDeletedAtIsNull` 교체 |
| **커밋** | `refactor: [PaymentService.java] 비관적 락 → Redis 분산락으로 재고 동시성 제어 전환` |
